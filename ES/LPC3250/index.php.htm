LPC3250 OEM Board Resources

Thank you for buying Embedded Artists' LPC3250 OEM Board based on NXP's ARM926EJ-S LPC32xx microcontroller family.

These support pages contains a lot of information and programs (sample applications) that will QuickStart your program development!

Please note that Embedded Artists does not provide support for the program code that can be downloaded from these support pages. The program code and sample applications have been created in order to ease application programs development on the LPC3250 OEM Board as well as getting up-and-running quickly.
Separate support agreements are available and subject to individual negotiation.

Please drop us an e-mail if you find any errors (including spelling errors) or confusing information. Suggestions for new/extended information sections are also welcome.

LPC3250 OEM Board Limitation Note (updated 2010-10-18)

Keil uVision 4.12

If sample applications stop to function with Keil uVision 4.12 disable the VFP setting in your project.
- Project -> Options for target -> Target
- Find the Floating Point Hardware setting in the Code Generation Box
- Change the setting from "In-line (strict ANSI)" to "Not used"

The reason is that VFP is not enabled in the MCU from the startup files used in the sample applications, but uVision 4.12 may generate code that require VFP support if the setting is "In-line (strict ANSI)" .

Restore kickstart and S1L Bootloaders

The LPC3250 OEM Board comes preloaded (stored in NAND flash) with the kickstart and S1L bootloaders. Use the application below to restore or update the bootloaders.

Important
If you have an S1L bootloader with older build date than Oct 4 2009, please use the restore application to update the bootloader. Running older bootloaders could result in an unstable application if the DDR SDRAM is used. The core voltage is slightly increased with the S1L bootloader dated Oct 4 2009 or newer.

You can see the S1L build date in a terminal application when S1L boots. 

  Embedded Artists LPC3250 board
  Build date: Oct  4 2009 12:16:26

Section 5.4.3 - Service Boot in the User's Manual describe how to download a bin file to the LPC3250 OEM Board.

Release notes
2010-10-15

  • Fixed a bug in the Ethernet driver that caused problems with 10Mbps HUBs
  • Updated Keil uVison project files so that VFP is not enabled by default. This caused problems with uVison 4.12 or later.
  • Updated the SPI clock frequency in the when using the SPI NOR flash. The old value was 1 MHz which doesn't work correctly with the Spansion NOR flash. The new value is 10 MHz
  • Fixed a bug in SPI driver. The limit for the SPI clock rate divisor was set to maximum 6 bits, but the correct limit is 7 bits.

2009-10-04
  • Clock frequency is now by default 266 MHz instead of 208 MHz.
  • Added functions in the startup code to control the 1.2V core voltage. When running at 266 MHz it must be at least 1.3V.

Problem with booting from NAND flash?
The NAND partition table has recently been updated by NXP. You need to modify the u-boot environment variables in order to update and boot from NAND flash (if you are using the latest Linux kernel and u-boot from LTIB).

Please verify the NAND partition offsets on NXPs community site: NAND partition schemes
